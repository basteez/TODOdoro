## Successes

- **Architecture validated again — zero production code changes for Story 4-3.** The event-sourced identity-based keying (`todoId`, not `title`) meant `TodoRenamedEvent` falls through to `default: return state` in `projectDevotionRecord`. Only 3 regression guard tests added. This is the third consecutive epic where the Epic 1 architecture investment has paid compound returns.
- **Proportional review flagging from Epic 3's retro worked.** Stories 4-2 and 4-4 were flagged in advance as complex. Code review caught real issues in both: missing popover integration tests (4-2) and a `computeTimeSpan` bug + AC #5 timing violation (4-4).
- **Pattern reuse is accelerating.** CompletionMoment extended with `variant` prop for seal display (not a new component). DevotionDots extended with `onClick` prop for popover trigger. Radix component patterns (Popover, Dialog) are consistent and predictable.
- **WCAG compliance caught proactively in Story 4-1.** Light theme `--devotion` contrast failure (~1.9:1) found during development, not in review. Fixed by darkening from `hsl(38,80%,58%)` to `hsl(38,80%,38%)` (~3.4:1 ratio).
- **Epic 3 retro commitments largely met.** Proportional review (✅), spec deviation discipline (✅), local pipeline discipline (no evidence of failures). Team accountability is improving.
- **Zero blockers, zero incidents, 100% story completion — third consecutive clean epic.**

## Challenges

- **Self-review still missing substantive issues.** Code review caught meaningful bugs in 2 of 4 stories (50%). Story 4-2: Task 6.3 marked done but popover integration tests never written. Story 4-4: `computeTimeSpan` using raw ms diff instead of calendar days, and seal event firing before CompletionMoment dismissed. This pattern has persisted since Epic 3.
- **Compressed same-day execution continues.** All 4 stories shipped on 2026-03-14 — same pattern as Epic 3 (all 7 stories on 2026-03-12). Rapid execution compresses review windows.
- **Boot sequence integration tests still unaddressed.** Carried since Epic 1, no progress across Epics 2, 3, or 4. Consciously deferred — Epic 6 will overhaul storage, making current tests throwaway work.

## Key Insights

1. **Architecture investment pays compound returns (again).** Story 4-3 is the clearest evidence yet — the system works exactly as designed, and regression tests are insurance, not fixes.
2. **Retro commitments are sticking.** Proportional review flagging is now a proven practice. Three of three Epic 3 action items were addressed.
3. **Self-review gaps are acceptable when code review catches them.** The safety net works. The team accepts this tradeoff — velocity over perfect self-review discipline.
4. **Story 4-4 gave Epic 5 a head start.** `handleSealTodo`, CompletionMoment seal variant, shelf projection wiring — all implemented ahead of Epic 5's Story 5-1.
