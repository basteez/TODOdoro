## Successes

- **The product is tangible.** Opening the app now delivers the spatial canvas experience envisioned in the PRD — full-viewport, dark theme, zero onboarding, create/rename/drag/zoom/pan. The product *feels* intentional.
- **Command handler pattern became a reliable template.** Established in Story 2.3 (`handleDeclareTodo`), reused cleanly in 2.4 (`handleRenameTodo`) and 2.5 (`handlePositionTodo`). Architecture paying dividends.
- **Test infrastructure set up early and used consistently.** vitest + jsdom + testing-library configured in Story 2.1, used across all subsequent stories. Test count grew from 161 to 280+.
- **Story validation worked.** All stories had thorough dev notes, architecture references, and project structure notes — a direct result of Epic 1 retro action item.
- **WCAG 2.1 AA accessibility built in from the start.** Focus rings, skip link, keyboard navigation, `aria-labels`, touch targets — not bolted on later.
- **Zero hard blockers, zero production incidents, 100% story completion.**

---

## Challenges

- **React Flow integration surprises in 4/7 stories.** `exactOptionalPropertyTypes` (2.3), `ResizeObserver` polyfill (2.1), right-click pan vs context menu (2.6), `nodesFocusable` double tab stop (2.7). Third-party library edge cases were the primary source of friction.
- **Missing test cases caught in code review in 4/7 stories.** Hint re-show (2.2), blur-unchanged-title (2.4), debounce behaviour (2.5), accessibility tests (2.7). These were not obscure edge cases — they were core behaviours.
- **Attention-to-detail gaps.** Specs were good; execution didn't always match. CSP header (2.1), touch targets (2.6), ErrorBoundary placement (2.1) — all specified in architecture/UX docs but missed during dev.
- **Agent model switched mid-epic (again).** Stories 2.1-2.3 used Opus 4.6, Stories 2.4-2.7 used Sonnet 4.6. This was an explicit Epic 1 retro commitment that was not followed.
- **Code review acting as a quality gate instead of a refinement pass.** Review was catching bugs that self-review should have caught first.

---

## Key Insights

1. **Self-review before submission is the biggest lever.** Most review fixes were things already in the spec — catching them ourselves saves entire iteration cycles.
2. **Follow through on retro commitments.** The model switch happened twice now. Commitments without accountability are just words.
3. **The command handler pattern is a proven accelerator.** Carry it forward into Epic 3.
4. **Refine boundary stories thoroughly.** React Flow integration surprises hit 4/7 stories. Durable session stories in Epic 3 carry the same kind of boundary risk.
