## Successes

- **Domain-first strategy delivered speed and quality simultaneously.** Pure functions with injected dependencies made everything trivially testable. The 100% coverage gate was an accelerator, not a bottleneck.
- **Architectural constraints held and paid off.** Zero production deps in domain, acyclic package graph enforced by TypeScript project references, error-as-value pattern, discriminated unions for state — all held throughout the epic.
- **Test utility patterns propagated cleanly.** `FakeClock`/`FakeIdGenerator` established in Story 1.3 reused in 1.4, extracted to shared `testUtils.ts` in 1.6, used through 1.7.
- **Fast and effective execution.** Stories 1.2, 1.4, and 1.6 were zero-debug passes. The architecture gave clear rails to run on.
- **CI pipeline configured and green.** Domain coverage hard-gate, typecheck, build — all automated from Story 1.7.

---

## Challenges

- **Story specs had internal inconsistencies.** Story 1.3 listed function signatures without `clock`/`idGenerator` params, but the event interfaces and architecture required them. Caught during dev, not planning.
- **Recurring review feedback across stories.** Defensive guards flagged in Stories 1.3, 1.4, 1.5. Missing branch tests flagged in 1.3, 1.6, 1.7. Configuration hygiene flagged in 1.3, 1.7. Same class of issue repeated.
- **Cross-boundary stories were harder to predict.** Scaffolding (1.1), coverage gap hunting (1.5), and boot sequence wiring (1.7) all had environmental surprises. Pure domain stories (1.2, 1.3, 1.4, 1.6) were smooth.
- **Deferred type refinement caused cascading updates.** Changing `SnapshotCreatedEvent.snapshotState` from `unknown` to `SnapshotState` in Story 1.7 broke 5 test files, requiring a `DUMMY_SNAPSHOT_STATE` helper.
- **Agent model switched mid-epic.** Stories 1.1-1.4 used Sonnet, Stories 1.5-1.7 used Opus. Introduced inconsistency.

---

## Key Insights

1. **Domain-first with 100% coverage was an accelerator, not a tax.** Speed and quality went hand-in-hand because pure functions with injected dependencies are trivially testable.
2. **Validate stories before dev.** Spec-vs-reality mismatches cost implementation time that a refinement pass would have caught cheaply.
3. **Complexity lives at boundaries, not in domain logic.** Scaffolding, CI, boot sequence were harder than aggregates and projections.
4. **Consistency matters.** One model, one approach, one epic.
