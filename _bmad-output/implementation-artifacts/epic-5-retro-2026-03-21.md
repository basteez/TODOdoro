## Challenges

- **Self-review still missing substantive issues — 3rd consecutive epic.** Code review caught issues in 80% of stories. Story 5.3 was missing 6 entire integration tests. Story 5.4 shipped with a non-functional slide animation (missing @keyframes definition). Pattern has persisted since Epic 3.
- **Compressed same-day execution — 3rd consecutive epic.** All 5 stories shipped on 2026-03-16. Same pattern as Epic 3 (7 stories on 2026-03-12) and Epic 4 (4 stories on 2026-03-14). Compressed execution means review findings from Story N cannot feed forward into Story N+1.
- **Visual regression slipped through.** Post-ship fix required for TodoCard title overlap (PR #19) — start-session button overlapping card title. Not caught by 404 automated tests or Playwright E2E.

---

## Key Insights

1. **Architecture investment pays compound returns (5th consecutive epic).** Story 5.1 is the most dramatic example — an entire story reduced to a single animation because Epic 4 pre-built the infrastructure.
2. **Compressed execution compresses learning.** When all stories ship in one day, review findings can't feed forward. Issues found in Story 3's review are already baked into Stories 4 and 5. One-story-at-a-time execution fixes this systemically.
3. **Code review is a reliable safety net but shouldn't be the primary gate.** 80% catch rate means the net works. But falling into it less often is the goal.
4. **Epic 6 needs research before execution.** WASM, Web Workers, OPFS, SQLocal, Drizzle — all new technology for the project. Technical research is a critical path item before story creation.
